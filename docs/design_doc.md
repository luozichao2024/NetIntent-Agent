# NetIntent-Agent 系统设计文档

## 1. 项目背景与目标
传统网络运维通常需要管理员手动编写和下发命令行配置，这在复杂的网络环境中效率低下且容易出错。本项目旨在通过引入大模型智能体（LLM Agent）技术，构建一个**意图驱动的网络自治运维系统 (NetIntent-Agent)**。
目标是让系统能够自动解析用户的自然语言需求，完成策略的自动生成、下发以及闭环验证，并在出现网络异常时实现自动重规划与自愈。

## 2. 核心创新点
本项目重点突出了以下几个关键技术创新：
1. **自然语言到网络策略的自动翻译**：利用 Intent Agent 准确提取源、目的、约束等关键要素，极大地降低了网络管理门槛。
2. **多 Agent 协同闭环**：通过 Intent, Planner, Executor, Verifier 四个 Agent 的分工与协作，系统化地解决了复杂网络任务的链式执行问题。
3. **遥测反馈驱动的策略重规划**：不仅停留在配置下发阶段，还引入了 Verifier Agent 主动探测网络状态。当实际延迟不符合要求时，系统能够基于反馈实现“自愈”。
4. **面向实验网场景的实用性**：底层支持生成真实的 `iptables`、`ovs-ofctl` 等命令，具备落地实验网的可行性。
5. **现代化的 GUI 可视化展示**：为用户提供了直观透明的黑盒内部运行日志和拓扑视图，方便追踪溯源。

## 3. 系统架构与模块设计
系统采用了流水线 (Pipeline) 式的处理架构。

- **Intent Agent**: 意图理解中心，将用户的非结构化文本翻译为结构化的 JSON 意图。
- **Planner Agent**: 策略大脑，根据意图 JSON 和网络拓扑生成最佳实施路径和底层配置策略。
- **Executor Agent**: 动作执行器，通过内置的白名单和防注入机制，安全下发网络命令。
- **Verifier Agent**: 持续监测器，使用 ping/iperf/tc 等工具对配置结果进行校验，如发现偏差则反馈给 Planner 触发修正。
