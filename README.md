# NetIntent-Agent：意图驱动网络自治运维系统

## 1. 项目简介

本项目构建了一个基于大模型智能体的网络自治运维 MVP 系统。用户只需通过自然语言输入网络需求，系统将自动完成：
**意图解析 (Intent) → 策略规划 (Planner) → 配置执行 (Executor) → 结果验证与遥测 (Verifier) → 链路异常自愈 (Re-planning)**
的完整网络自治闭环。

本项目重点展现了：
1. **自然语言到网络策略的自动翻译**。
2. **多 Agent 协同闭环架构**。
3. **基于遥测反馈驱动的策略自动重规划与网络自愈**。

## 2. 核心演示场景

- **场景 1：访问控制自动生成与验证**
  - **输入**：“禁止学生网段访问管理服务器，但允许教师网段访问。”
  - **流程**：自然语言解析 → 生成 ACL 规则 → 下发 iptables 规则 → 验证连通性。
- **场景 2：链路拥塞检测与自愈**
  - **输入**：“保证客户端到服务器的延迟低于50ms。”
  - **流程**：解析延迟约束 → 选择最优路径 → 下发流表 → 验证网络延迟；若发现延迟超标或链路故障（模拟拥塞），系统会自动检测并重新规划备用路径（网络自愈）。

## 3. 环境配置与启动说明

本项目提供了一键启动的 Demo 和交互式可视化 GUI。评委老师可以按照以下步骤轻松体验完整流程。

### 3.1 依赖安装

推荐运行环境：Python 3.8 及以上。

```bash
# 安装所需依赖
pip install -r requirements.txt
```

### 3.2 启动可视化控制台 (推荐体验)

我们提供了一个基于 Streamlit 的现代化 Web 控制台，直观展示多 Agent 协同和底层执行过程：

```bash
streamlit run app.py
```
> 启动后，请在浏览器中访问 http://localhost:8501 即可通过可视化界面体验“一键输入，自动配网”的完整闭环。

### 3.3 命令行测试模式 (CLI)

如需在命令行下快速测试主干流程，可使用以下命令：

**测试访问控制场景：**
```bash
python main.py --scenario access
```

**测试低延迟/自愈场景：**
```bash
python main.py --scenario latency
```

**运行单元测试：**
```bash
pytest
```

## 4. 系统核心架构

为了实现网络的自动化闭环运维，我们设计了以下核心 Agent 模块：
- **Intent Agent**: 解析用户的自然语言意图，精准识别源、目的、动作和约束条件。
- **Planner Agent**: 结合全局网络拓扑和意图，规划最优的网络策略组合。
- **Executor Agent**: 作为底座执行器，将高维策略翻译为具体的底层设备配置命令（支持白名单机制以确保安全）。
- **Verifier Agent**: 负责闭环校验，通过网络遥测工具监测实际运行状态。一旦检测到偏离预期的异常（如延迟过高），立刻触发重新规划。
